Backend Notisability — listo
Stack: Node.js + Express + TypeScript, Prisma ORM + PostgreSQL, JWT (access/refresh), Mercado Pago Checkout Pro + webhooks.

Estructura (packages/backend/)
prisma/schema.prisma — 14 modelos relacionales (Usuarios, Roles, Categorías, Productos, ProductFile, Cart/CartItem, Orders/OrderItem, Payments, UserLibraryItem, Reviews, Promotions/ProductPromotion)
prisma/seed.ts — siembra roles, categorías y usuario admin (admin@notisability.com / Admin123!)
src/modules/ — 11 módulos (auth, users, roles, categories, products, cart, orders, payments, library, reviews, promotions), cada uno con .dto.ts / .service.ts / .controller.ts / .routes.ts
src/common/ — middlewares (auth, validate, errorHandler), errores, utils (jwt, password)
src/config/ — env, prisma client, mercadopago client
src/app.ts / src/server.ts
Para arrancar

cd packages/backend
npm install
cp .env.example .env   # configurar DATABASE_URL, JWT secrets, MP_ACCESS_TOKEN, etc.
npx prisma migrate dev --name init
npx prisma db seed
npm run dev
Endpoints clave (/api/...)
auth: register, login, refresh, logout
users: perfil propio, cambio de contraseña, gestión admin
categories, products: catálogo público + gestión por proveedor/admin, subida de archivos versionados
cart, orders: carrito → orden con descuentos por promoción
payments: preferencia de Mercado Pago + webhook con verificación de firma x-signature
library: biblioteca digital del usuario + descarga de archivos
reviews, promotions: reseñas (requieren compra) y promociones con códigos
npx tsc --noEmit compila sin errores. Cualquier ajuste adicional (frontend, tests, despliegue) lo retomamos cuando quieras.
